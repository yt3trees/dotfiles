# ワークフロー詳細(6ステップ)

## Step 1: 目的確認

- `input/` フォルダの全ファイルを読み込む
- ターゲット(誰に向けたプレゼンか)を抽出
- ゴール(何を達成したいか)を抽出
- 期待アクション(聞き手に何をしてほしいか)を抽出
- 不明な点があればユーザーに質問する

## Step 2: 構成設計

1. キーメッセージの抽出
   - 元資料から最も伝えたいポイントを特定
   - 各スライドに1つのキーメッセージを割り当て

2. 情報グルーピング
   - 関連する情報をまとめてスライド単位に整理
   - 1スライド1メッセージの原則

3. 論理構造の選択
   - PREP(結論→理由→具体例→結論)
   - 問題解決(課題→原因→解決策→効果)
   - 比較提案(現状→選択肢→比較→推奨)
   - 時系列(過去→現在→未来)

4. 各スライドの役割決定
   - 表紙、目次、コンテンツ、まとめ等の役割を明確にする

## Step 3: レイアウト選択

- 各スライドに15パターンから最適レイアウトを割り当て
- layout-selection-guide.md の2ステップフローチャートに従う
- 迷った場合は bullet_points をデフォルトとする

## Step 4: テキスト調整

- character-limits.md の制限に合わせてテキストを調整
- 超過時は以下の優先順位で対応:
  1. 要約して収める
  2. スライドを分割する
  3. 末尾を「...」で切り詰める
- キーメッセージは必ず述語で終える(体言止め禁止)

## Step 5: slides.md 出力

- 専用マークダウン記法で `output/slides.md` に出力
- layout-rules.md の記法に厳密に従う
- 各スライドは `---` で区切る
- レイアウト指定は `<!-- layout: xxx -->` コメントで行う

## Step 6: PowerPoint 生成

Pythonスクリプトを実行して PPTX を生成する。

```bash
cd .claude/skills/pptx-generator/scripts
python slide_generator_pptx.py \
  --config ../config.json \
  --markdown-file ../../../../output/slides.md \
  --output-dir ../../../../output
```

### CLI オプション

| オプション | 説明 | 必須 |
|-----------|------|------|
| --markdown-file | slides.md のパス | Yes |
| --config | config.json のパス | No |
| --title | プレゼンテーションタイトル | No |
| --output-dir | 出力先ディレクトリ | No |
| --template | テンプレート.pptx のパス | No |
| --layout | 特定レイアウトのみ出力(カンマ区切り) | No |
